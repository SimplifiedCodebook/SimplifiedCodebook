Ques 5 : How to configure authentication to create custom user and password in spring security?

  • Using application.properties file [Not recommended]

Spring.security.user.name = demo
Spring.security.user.password = demo

  • Spring won’t create default user and password now.
  • To provide custom configuration, configure AuthenticationManager.
  • For that, the older way was to create a configuration class which extends WebSecurityConfigurationAdapter and override the configure method. But WebSecurityConfigurationAdapter is deprecated now.
  • Spring security now expects to create a bean for SecurityFilterChain to configure auth security.
  • Also spring security expects an instance of “PasswordEncoder” to encrypt the password, so create beans for the same. For testing, use “BCryptPasswordEncoder”.
  • Also use @EnableWebSecurity annotation to tell spring this class contains web security configuration.
  • Create a demo controller for endpoints for normal and admin users.

@RestController
@RequestMapping("/demo")
public class DemoController
{
  @GetMapping("/normal")
  public ResponseEntity<String> normalUser()
  {
     return ResponseEntity.ok("welcome normal user..");
  }


  @GetMapping("/admin")
  public ResponseEntity<String> adminUser()
  {
     return ResponseEntity.ok("welcome admin user..");
  }

  @GetMapping("/public")
  public ResponseEntity<String> publicUser()
  {
     return ResponseEntity.ok("welcome public user..");
  }

}
